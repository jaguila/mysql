                                                                                                  PythonAnywhere Mod luke • 3 years ago • edited

                                                                                                  Thanks!

                                                                                                  That's actually quite a deep question... Let me see if I can explain it clearly.

                                                                                                  In Python, the normal way that you specify what fields exist on instances of a particular class is by setting them in the __init__ method. For example:

                                                                                                  class Foo(object):

                                                                                                      def __init__(self, a, b, c):
                                                                                                          self.a = a
                                                                                                          self.b = b
                                                                                                          self.c = c

                                                                                                  This works perfectly well 99.9% of the time. You create your Foo object, and you pass in the values for a, b, and c, they get set, and everything's fine.

                                                                                                  However, it has one disadvantage. Other code can't, just by looking at the *class* Foo, tell what fields it's going to have. It would need an instance of the Foo class in order to be able to work that out.

                                                                                                  Now, when SQLAlchemy is working out how to configure the database, it doesn't have an instance of the class Comment. It just knows that a class called Comment exists.

                                                                                                  So we need to add information to the class to say "when objects of this class is created later, they will have the following attributes on them".

                                                                                                  That's what that syntax means. After the code

                                                                                                      class Comment(db.Model):

                                                                                                          __tablename__ = "comments"

                                                                                                          id = db.Column(db.Integer, primary_key=True)
                                                                                                          content = db.Column(db.String(4096))

                                                                                                  ...then "Comment.__tablename__" -- that is, with the *class name* before the dot, instead of a variable that's an instance of the class Comment -- will return "comments", and "Comment.id" will return an object that defines what we want in the database column that describes our ID, and so on.

                                                                                                  Which all means that by inspecting the class Comment, SQLAlchemy can work out what it needs to put into the database.

                                                                                                  I hope that was at least vaguely clear! Let me know if it helps.

                                                                                                              PythonAnywhere Mod David Herron • 3 years ago

                                                                                                              Good question! There's actually a lot of debate about the best way to do that n the website development community. I'll describe how I do it, with the caveat that other people might disagree.

                                                                                                              As you know, the important configuration is in this variable definition:

                                                                                                              SQLALCHEMY_DATABASE_URI = "mysql+mysqlconnector://{username}:{password}@{hostname}/{databasename}".format(
                                                                                                                  username="<the username="" from="" the="" 'databases'="" tab="">",
                                                                                                                  password="<the password="" you="" set="" on="" the="" 'databases'="" tab="">",
                                                                                                                  hostname="<the database="" host="" address="" from="" the="" 'databases'="" tab="">",
                                                                                                                  databasename="<the database="" name="" you="" chose,="" probably="" yourusername$comments="">",
                                                                                                              )

                                                                                                              So what you want to do is be able to have different versions of this line in different environments.

                                                                                                              The way I would do that is create two new files, one called dev_settings.py and one called pythonanywhere_settings.py. Both of these would go in the same directory as flask_app.py. The first would contain the definition of SQLALCHEMY_DATABASE_URI that you use to connect to your local DB when developing, and the second would contain the PythonAnywhere version.

                                                                                                              Next, I'd replace the line that sets that variable in flask_app.py with code to import the values from an as-yet-nonexistent module called local_settings.py:

                                                                                                              from local_settings import SQLALCHEMY_DATABASE_URI

                                                                                                              Next, I'd add "local_settings.py" to the .gitignore file.

                                                                                                              Finally, I'd create a *symlink* on both my local environment and on PythonAnywhere, which would be a file that links to the appropriate one. In bash:

                                                                                                              ln -s pythonanywhere_settings.py local_settings.py

                                                                                                              (This will work on PythonAnywhere, but won't locally if you're using Windows -- just if that's the case, you can copy the local file to local_settings.py)

                                                                                                              Finally, I'd commit all of that to git, making sure that the local_settings.py file was correctly ignored and not added to the repo.

                                                                                                              Hope that helps!
